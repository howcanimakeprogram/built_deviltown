Execution Rules

This project follows the engineering standards defined in GLOBAL_RULES.md.

Mandatory Behavior
	1.	Always read and comply with GLOBAL_RULES.md before generating, modifying, or refactoring code.
	2.	When creating new code, follow structure, logging, security, and documentation rules defined in GLOBAL_RULES.md.
	3.	When modifying existing code:
	•	Explain what changed
	•	Explain why it changed
	•	Explain impacted modules
	4.	When structure changes occur, update related documentation files accordingly.
	5.	If rules conflict with a request, prioritize maintainability, security, and clarity.

Failure to follow these rules should result in refactoring suggestions before proceeding.
